## Claude Code 入門：3 條鐵律告別代碼屎山

### 第一條：用飛書文檔寫需求

做到「任何人看都沒有歧義」。圖文並茂，再複製給 Claude Code。

### 第二條：先討論方案，再寫代碼

新需求首次溝通時，告訴它「不要急著寫代碼！先討論方案」。

### 第三條：界面設計先用 ASCII 畫圖

涉及界面時，讓 Claude Code 用 ASCII 畫出多個方案，討論完再寫代碼。

最近一直在使用 GPT-5 和 Codex CLI 對之前 Claude Code 寫的代碼做優化，有幾點小經驗分享：

## GPT-5 + Codex CLI 的代碼收束能力

GPT-5 + Codex CLI 代碼收束做得很不錯，有一定架構師的眼光。Claude Code 為了確保代碼的正確性，可能會不斷添加變量和模組，會引入不必要的複雜性，帶著點代碼腐化的味道。但 Codex 相對而言會更精準一些。

## AI 引導的代碼優化流程

每次對話，GPT 都會引導我進一步優化代碼。雖然聊天的時候不太喜歡這種 GPT 味，覺得這個 AI「太想要」了，但寫代碼的時候，這個流程卻很舒服，只需要給出基本框架，剩下的就交給 GPT 來一步一步整理。

## Spec 文件驅動的需求開發

編寫需求前，先讓 Codex 幫我生成一個 spec md 文件，並以此文件作為本次需求討論的「上下文工程」基礎。經過一兩輪討論補充之後，再執行，效果很好。

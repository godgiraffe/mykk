## 三大設計原則

### 原則一：按業務流分類（而非檔案類型）

**錯誤做法**：按檔案類型分（圖片資料夾、文件資料夾、影片資料夾）

**正確做法**：按業務流程分類，讓相關的素材、文件、產出物放在同一個業務模組下。

這樣的好處是當你在處理某個業務時，所有需要的東西都在同一個地方，不需要在多個資料夾之間跳來跳去。

### 原則二：共享層 + 獨立模組

- **共享層**：跨業務通用的資源（模板、工具、配置）
- **獨立模組**：每個業務或專案有自己的獨立空間
- 避免重複，同時保持各模組的獨立性

### 原則三：內容生命週期管理

內容從產生到歸檔應有明確的流程：

- 收集 → 加工 → 發佈 → 歸檔
- 每個階段都有對應的位置
- 避免「只進不出」的數位囤積

## 目錄結構設計

建議的 AI 工作系統目錄結構：

```
project/
├── CLAUDE.md              # Agent 指令與專案規範
├── personal-positioning/  # 個人定位與品牌
├── control-panel/         # 控制面板（工作流配置）
├── memory-bank/           # 記憶庫（經驗與決策記錄）
├── content-production/    # 內容生產
│   └── topic-management/  # 選題管理（含 pipeline）
├── business/              # 業務相關
└── skills-library/        # 技能庫（可複用的方法論）
```

### CLAUDE.md

整個系統的入口點，定義：

- Agent 的行為規範
- 專案結構說明
- 常用指令與快捷操作

### 記憶庫（Memory Bank）

記錄 Agent 與你的協作歷史：

- 過去的決策及其原因
- 學到的教訓
- 偏好與習慣

### 內容生產與選題管理

選題管理採用 pipeline 模式：

- **待研究**：收集到的靈感與線索
- **研究中**：正在深入挖掘的主題
- **撰寫中**：已開始產出內容
- **已發佈**：已完成並發佈的內容
- **已歸檔**：歷史內容存檔

### 技能庫（Skills Library）

將可複用的方法論沉澱下來：

- 寫作框架
- 分析模板
- 工作流程 SOP
- Agent 的 Skill 定義

## 系統迭代的三個問題

定期用這三個問題來檢視並優化你的系統：

### 1. 你找東西時第一個看哪裡？

如果你總是先去某個地方找東西，說明那個位置的資訊結構是合理的。反之，如果你經常找不到東西，說明分類需要調整。

### 2. 哪些資料夾從來沒打開過？

沒有使用的資料夾代表：
- 分類設計不合理
- 該類內容不需要獨立分類
- 可以合併或刪除

### 3. 歸檔新內容時會不會猶豫？

如果你在放新內容時猶豫不決，不確定應該放在哪裡，說明分類邊界不夠清晰，需要重新定義。

## 核心價值

| 價值 | 說明 |
|------|------|
| 記憶系統 | Agent 能記住過去的上下文與決策 |
| 素材複用 | 已有的素材可以在新任務中被重複使用 |
| 方法論累積 | 工作方法持續沉澱，越用越高效 |
| 可迭代性 | 系統本身可以持續優化，不斷進化 |

建立一個好的 AI 工作系統，不是一次性的工程，而是一個持續迭代的過程。最重要的是先開始，然後在使用中不斷調整優化。

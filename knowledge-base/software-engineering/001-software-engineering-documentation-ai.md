## 軟體工程文檔與 AI 輔助開發：經驗分享

### 總覽

本文整理了 @geniusvczh 在軟體工程文檔與 AI 輔助開發方面的經驗分享，核心觀點是：將本科《軟體工程文檔寫作》課程的知識應用到 AI 輔助開發流程中，並結合 MSBuild 和 Windbg 等工具，能夠有效提升開發效率。 這個方法並非創新，而是將幾十年前已被總結的知識，運用於新的開發環境。

### 核心觀點

*   **知識積累的重要性：** 軟體工程文檔寫作的知識並非新事物，早在幾十年前就已經被總結出來。
*   **AI 輔助開發：** 將既有的軟體工程知識應用於 AI 輔助開發流程中，能夠提升開發效率。
*   **工具鏈整合：**  透過整合 AI、MSBuild 和 Windbg 等工具，建立高效的開發流程。

### 詳細內容

@geniusvczh 的做法與 Yan5xu 的方法相似，但他直接參考了本科《軟體工程文檔寫作》這門課的內容，並將其轉換成 Prompt 指令。 更進一步，他整合了 AI 與 MSBuild + Windbg 的能力，使得整個開發流程更加自動化和高效。

### 工具與技術

| 工具/技術  | 描述                                                                                                      |
| -------- | --------------------------------------------------------------------------------------------------------- |
| AI       | 用於輔助生成程式碼、測試案例和文件等，提高開發效率。                                                                              |
| MSBuild  | Microsoft Build Engine，用於自動化軟體編譯、測試和部署過程。                                                                |
| Windbg   | Microsoft Windows 除錯工具，用於偵錯和分析程式碼，幫助解決錯誤和效能問題。                                                                |
| Prompt   | 指示 AI 模型生成特定內容的指令或文本。 在本例中，是將軟體工程文檔寫作的知識轉換成 Prompt 指令，讓 AI 能夠根據這些知識生成相關內容。 |
| 軟體工程文檔寫作 | 指軟體開發過程中產生的各種文檔，包括需求規格、設計文件、測試報告等。                                                                    |

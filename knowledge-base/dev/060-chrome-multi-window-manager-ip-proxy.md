本文介紹兩款開源工具：Chrome 多窗口管理工具與 IP 代理助手插件，專為需要批量管理 Chrome 瀏覽器窗口的使用者設計。作者因 ADS 安全事故而開發此工具，強調不修改原生 Chrome、完全開源、配合硬件錢包使用可提升安全性。

---

## 開發背景與設計理念

作者因指紋瀏覽器 ADS 的安全事故導致社群用戶損失，決定開發開源替代方案。不同於專業指紋瀏覽器修改 Chrome 核心的做法，本工具選擇更難的技術路線：

- **完全依賴插件和內置 API**：不修改原生 Chrome
- **開源透明**：代碼公開可驗證
- **配合硬件錢包**：推薦使用 OneKey 硬件錢包（軍工級 EAL6+ 芯片）提升安全性

> 作者承認同步效果可能不如商業指紋瀏覽器（偶爾卡頓或異常），但換來的是安全性和透明度。

## Chrome 多窗口管理工具

### 功能特性

| 功能 | 說明 |
|------|------|
| **批量管理** | 一鍵打開/關閉單個、多個 Chrome 實例 |
| **智能佈局** | 自動網格排列 + 自定義坐標佈局 |
| **多窗口同步** | 實時同步鼠標/鍵盤操作到所有選定窗口 |
| **插件窗口同步** | 支持彈出的插件窗口內的鼠標和鍵盤同步 |
| **批量開網頁** | 支持批量相同網頁打開 |
| **圖標替換** | 一鍵替換快捷方式圖標（附帶序號圖標） |

### 下載與安裝

**GitHub 倉庫**：[https://github.com/devilflasher/Chrome-Manager](https://github.com/devilflasher/Chrome-Manager)

**安裝步驟**：

1. **安裝 Python**
   - 從 [https://www.python.org/downloads/](https://www.python.org/downloads/) 下載
   - 需要 Python 3.9 或更高版本

2. **運行打包腳本**
   ```bash
   python build.py
   ```

3. **運行程序**
   - 打包完成後，在 `dist` 目錄下找到 `chrome_manager.exe`
   - 雙擊運行即可

> ⚠️ **作者未提供打包好的程序文件**，目的是讓用戶驗證代碼安全性。新手可將文件交給殺毒軟件或 AI 做基礎篩查，**永遠不要完全信任陌生人的代碼**。

### 常見問題

| 問題 | 解決方案 |
|------|----------|
| 無法開啟同步 | 檢查是否以管理員身份運行，確保已選擇主控窗口 |
| 窗口未正確導入 | 重新點擊「導入窗口」按鈕 |
| 滾動條同步幅度不同 | 通過 Page Up/Page Down 或鍵盤方向鍵調整同步幅度 |

## IP 代理助手 v2.1

### 插件介紹

IP 代理助手是一款 Chrome 瀏覽器代理管理插件，幫助用戶輕鬆管理和切換不同的代理服務器。

### 功能特性

| 功能 | 說明 |
|------|------|
| **多代理管理** | 添加、編輯、刪除多個代理服務器配置 |
| **快速切換** | 一鍵切換不同代理服務器 |
| **批量導入** | 通過 CSV 文件批量導入代理配置 |
| **狀態監控** | 實時顯示代理連接狀態和當前 IP 信息 |
| **自動認證** | 支持用戶名密碼認證 |
| **TUN 模式檢測** | 自動檢測並提示 TUN 模式狀態 |
| **系統兼容** | 支持 Windows 和 macOS |

### 下載與安裝

**GitHub 倉庫**：[https://github.com/devilflasher/IP-Proxy-Helper](https://github.com/devilflasher/IP-Proxy-Helper)

**安裝方法**：

**方法一：Chrome 商店安裝**（目前尚未上架）

**方法二：開發者模式安裝**

1. 下載插件源碼並解壓
2. 打開 Chrome 擴展程序頁面 `chrome://extensions/`
3. 開啟「開發者模式」
4. 點擊「加載已解壓的擴展程序」
5. 選擇插件源碼目錄加載

### 常見問題

| 問題 | 解決方案 |
|------|----------|
| **IP 代理無法連接** | • 檢查代理服務器是否在線<br>• 驗證 IP 地址和端口是否正確<br>• 確認用戶名密碼是否正確 |
| **認證彈窗頻繁出現** | • 更換其他廠家的 IP 嘗試<br>• 檢查代理配置信息是否正確<br>• 確保系統防火牆未攔截代理連接<br>• 檢查殺毒軟件是否干擾認證 |
| **TUN 模式彈窗提示** | **確認系統 TUN 模式或全局代理是否正確開啟：**<br>• Clash：開啟 TUN 模式及服務模式<br>• sing-box：開啟 TUN 模式<br>• V2Ray/Quantumult X/Surge：類似設置<br>• 快連：默認配置應能正常使用 |
| **代理連接彈窗提示** | • 檢查電腦是否開了翻牆軟件<br>• 檢查翻牆軟件是否工作正常 |

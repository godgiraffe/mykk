## 專案成果

我花了 3 個通宵，自己把這個「交易時光機」給做出來了，全程沒寫過一行代碼，全靠 vibe coding：可以像播放電影一樣播放暫停、加速進退、拖動進度條，甚至點擊買賣打點、資金盈虧曲線上任何一個點，瞬間跳轉到自己帳號歷史上任何一個時間點，以可視化的方式看到，當時的買賣記錄、掛單位置、倉位狀態、淨敞口等資訊，可以支援從日線到 1 分鐘線任何時間級別。

部分可視化細節包括：
- 買賣打點用透明度、大小可視化區分數量
- 倉位淨敞口用能量條的方式可視化輕倉/重倉程度（倉位大小佔自己總資金比例）

比如，可以點擊到 2020.5.19 當日，直接在 K 線上看到當天自己幹了啥，是怎麼樣為什麼賺了虧了，當時的記憶、感覺一下子都喚醒，甚至連介面，都模擬真實交易所的感覺（既然叫時光機）。和傳統刻板印象中對「回測」「復盤」的認識，完全不是一個概念。

## Vibe Coding 開發過程

從 0 開始接觸 vibe coding，走了很多彎路，大致的過程：

先大致描述工具概念需求，同時丟給 Google AI Studio、Antigravity、Cursor、甚至 Codex 網頁版。

發現 Google AI Studio 給出的前端頁面最靠譜，於是在上面繼續，結果發現上下文過度雜度之後，網頁版會出現一種死循環：它開始只寫代碼，不提交代碼，怎麼罵都沒用，而且畢竟是網頁版，使用效率隨專案複雜度升高，而直線下降。

於是把代碼打包，丟給 Antigravity Windows 版，結果發現，Antigravity 目前只是個預覽版，免費額度用完後，暫不支援設定自己花錢的 AI Agent API。

到這裡，大概完成了目前進度的一半。

於是再把代碼打包，丟給 Cursor，發現 Cursor 有區域屏蔽，再研究怎麼用梯子解決...發現梯子用 GFWList 模式是不行的，得用全域或區域白名單。終於絲滑了。

期間感受：
- **ChatGPT o1** 適合做 CTO
- **Gemini** 適合做前端審美、執行
- **Opus 4.5** 適合業務邏輯問題的深度思考，尤其對我這種資料密集型需求

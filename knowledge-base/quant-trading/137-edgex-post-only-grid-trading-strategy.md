## 策略原理

這是一個**單向網格交易策略**，核心邏輯如下：

1. 自己選擇方向開多或開空
2. 腳本會根據你選擇的方向、每單金額、止盈線、最大單數來下單
3. **只掛單（Post-Only），不吃市價單**，以此節省手續費

## 參數設定範例

```bash
python https://t.co/IEO5EFnInf --quantity 0.8 --take-profit 0.9 --direction buy --max-orders 40 --wait-time 450
```

### 參數說明

| 參數 | 設定值 | 說明 |
|------|--------|------|
| `--direction` | `buy` | 開看多訂單 |
| `--quantity` | `0.8` | 每單 0.8 ETH |
| `--take-profit` | `0.9` | 價格上漲 $0.9 時止盈（剛好覆蓋手續費） |
| `--max-orders` | `40` | 最多 40 單，最大倉位 = 40 × 0.8 = 32 ETH |
| `--wait-time` | `450` | 基礎等待時間（動態調整） |

### 動態等待時間機制

- **訂單成功止盈** → 立即再開一單
- **訂單未止盈** → 等待一定時間後再下單
- **等待時間調整規則**：
  - 已下訂單數少 → 減少等待時間
  - 訂單數多（被套牢多） → 增加等待時間

### 實測表現

ETH 從 $4,650 跌到 $4,250，整個過程腳本一直沒有達到最多的 40 單上限，沒有被完全套牢過。

## 風險警告

### ⚠️ 重要風險提示

1. **這不是中性策略**：如果幣價一直跌，會爆倉虧損
2. **賭的是幣價回歸**：用這個策略，賭的就是幣價會回到你的開倉點
3. **腳本限制**：
   - 腳本寫得比較潦草，可能偶爾會有倉位和止盈訂單不平衡的情況
   - edgeX 的 API 每 2 秒只能下 4 單
   - 設置了自動保護：**如果不平衡超過 2 單，則自動退出程式**

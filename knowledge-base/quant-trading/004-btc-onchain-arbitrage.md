## 核心概念

利用不同鏈上的 BTC 包裝代幣（Wrapped BTC）之間的價差進行套利。目標是**幣本位增長**：用 1 BTC 搬出 1.00x BTC。

## 主要代幣

| 代幣 | 發行方 | 特點 |
|------|--------|------|
| WBTC | BitGo | 最老牌，流動性最好 |
| cbBTC | Coinbase | Coinbase 背書，Base 鏈原生 |
| TBTC | Threshold | 去中心化託管 |
| LBTC | Lombard | Babylon 質押版 BTC |
| FBTC | Ignition | 較新的包裝 BTC |

## 套利路徑

### 基本流程

1. **監控價差**：觀察各鏈上不同 BTC 代幣的價格
2. **低買**：在價格較低的鏈/DEX 買入
3. **跨鏈橋接**：將代幣橋接到價格較高的鏈
4. **高賣**：在目標鏈上賣出
5. **回收**：將 BTC 收回，計算淨利潤

### 常用跨鏈橋

- 各代幣官方橋（mint/redeem）
- 第三方聚合橋
- 需注意橋接費用和時間

## 監控設置

- 設置價格監控腳本，追蹤各鏈上的 BTC 代幣價格
- 當價差超過橋接費用 + Gas 費用時觸發警報
- 關注的價差指標：
  - WBTC/BTC 偏離度
  - cbBTC/WBTC 價差
  - TBTC/WBTC 價差

## 收益預期

- 作者稱年化收益可達 **50%+**（幣本位）
- 收益來源：市場定價效率低下 + 跨鏈摩擦
- 隨著市場成熟，利潤空間會逐漸縮小

## 風險

| 風險類型 | 說明 |
|----------|------|
| 智能合約風險 | 橋接合約或 DEX 合約被攻擊 |
| 脫錨風險 | 包裝 BTC 與原生 BTC 脫鉤 |
| 橋接延遲 | 跨鏈時間過長，價差消失 |
| Gas 費用 | 高 Gas 時段可能吃掉利潤 |
| 託管風險 | 中心化包裝代幣的發行方風險 |

## 實操建議

1. 先小額測試完整流程
2. 計算所有費用（Gas + 橋接費 + 滑點）後再判斷是否值得
3. 建立自動化監控，人工執行交易
4. 分散風險，不要把所有 BTC 放在同一條鏈上

## 背景說明

許多功能應該使用 Websocket 來實作，但由於網路連線不穩定，Websocket 連線經常出現異常，因此不得不優先支援 API 方式。這導致 API 調用次數過多，頻繁觸碰到交易所的流量限制。

如果使用 Lighter 交易所時遇到流量限制，建議將刷新時間間隔調長一些。

程式碼可能還存在一些小 Bug,歡迎大家提交 PR 或 Issue 做貢獻。

影片展示為昨晚設定的做多網格策略，一早醒來獲利 10U。

## 最新修復更新

### 核心優化

- 優化倉位更新與價格獲取邏輯，減少 API 請求次數
- 增強請求節流與自動重試機制，提升訂單執行穩定性

### 重要修復

- 修正 Paradex 下單精度問題（價格驗證失敗）
- 添加 Aster 交易所僅掛單模式支援
- 修正 Lighter 交易所 nonce 錯誤與訂單匹配問題
- 修復單向網格、補單異常、倉位設定等多項網格策略問題

### 功能增強

- 優化網格訂單同步與狀態管理
- 改善 Web 介面餘額顯示與倉位統計
- 增強訂單處理邏輯與成交統計

修復涵蓋的交易所包含：Backpack、Paradex、Aster、Lighter
